Installing Software Under Linux

Using RPM

RPM is the RPM Package Manager used to manage the installation of software on a variety of Linux distributions. More information on rpm can be found at www.rpm.org. What follows is a brief overview of the types of commands you will need to use to get the CCP1 GUI working.

Querying a package

To check if you have a package installed open a shell and enter:

rpm -q <package>

where <package> is the name of the package, for example 'python' for Python, 'tk' for Tk, 'tcl' for Tcl and 'vtk' for VTK. If you are not sure what the package is called, or are seraching for packages relating to a particular package, try the command:

rpm -qa | grep -i '<name>'

This queries the entire database of installed packages outputing a line for each package that contains the string <name>.

The output of these commands will not only tell you if you have the package installed, but what version you are using. This assumes the packages have been installed as rpm files - if you've installed them from source the rpm database will have no record of them.

Installing an rpm package

Sofwtare distributed as rpm packages are optimised for various linux distributions and machine architechtures, so you will need to know what sort of processor you are running. To find this out, open a shell and type the command:

uname -ps

This will display the version of the operating system you are using and the type of processor (e.g intel/pentium: i386, i586 etc), alpha etc.). Rpm packages for a particular architecture usually have suffices such as i386.rpm or alpha.rpm so you can select the correct one.

Download the rpm package for your system/architecture to a directory on your harddrive (it isn't too important where you save the file, as the rpm program that installs the program will copy all of the relevant files to the correct directories on your hard drive, allowing you to delete the rpm file afterwards if desired).

To install the rpm package, open a shell and cd into the directory where you downloaded the file. You will probably need to be root to have the relevant priviledges requried to run the installation, so type 'su' at the command prompt and enter your root password. As root enter:

rpm -ivh <package_name>

where <package_name> is the rpm file you downloaded. You should see a series of hash marks (####) indicating the progress of the installation and a message telling you if the installation was succesful.

Untarring and unzipping archives.

Many software packages are distributed as tarred and zipped archives. If a package has the suffix .tar, it indicates that it is a tar archive (several files and/or directories bundled into a single file). An additional bz or gz suffix (e.g. .tar.gz or .tar.bz) indicates that the package has been compressed with the gzip (gz) or bzip2 (.bz) utilities.

If a package (<package_name>) has been compresed with gzip or bzip, cd into the directory where the package resides and enter:

gzip -d <package_name> or gunzip <package_name>

if the file has been gzipped (.gz) or:

bzip2 -d <package_name> or bunzip2 <package_name>

if it has been bzipped. This will decompress the file and leave you with a file ending with a .tar suffix. To untar the file and create the directory type:

tar -xvf <package_name>

Alternatively, if your version of tar supports it, you can accomplish both steps in one go by typing:

tar -xvzf <package_name>

for a gzipped archive or:

tar -xvjf <package_name>

for a bzipped package.
