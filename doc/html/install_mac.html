Installation of the packages for the CCP1GUI under Mactintosh

Base Packages

Unfortunately, the version of Python that ships with OSX (Darwin at any rate) does not include Tk/Tcl or Tkinter by default, so you will need to install these packages first.

Tcl/Tk Aqua

Tcl/Tk for MacOSX can be downloaded from Apple - just follow the instructions for installing them.

Tkinter ( Numeric Python and IDLE )

The next step is to install the Mac Python add-ons, which include a Package Manager tool that you can use to install Tkinter ( as well as Numeric Python and IDLE ). You can get the Package Manger from MacPython website. Once you have installed this, it will create a folder call MacPython-version ( where version is the version of Python you are using - e.g. 2.3 ) in your Applications folder. Use the Finder to navigate to this folder and then run PackageManager program.

NB: If this crashes on opening with a message about being unable to open a ".plist" file, then the Package Manager is looking at the wrong online database to get it's list of available packages. To find out about current databases, visit the packman webpage and get the url for a suitable database. Then in PackageManager, select File -> Open Url... and paste in the url.

In the package manager then select the following packages:

  • Numeric-version-binary
  • _tkinter-version-binary
  • IDLE-version-binary

(where version will be a different number depending on the version) and then click the install button at the bottom of the page to install the packages.

If the above commands work for you, you should now be in a position to download one of the software bundles. Only read on if you need to install the additional packages yourself.

Additional Packages

Scientific Python

The Scientific Python Website has links for Mac installers that you can download. If these aren't available or you think installers are for wusses and you crave the command-line, then you can follow the instructions for installing Scientific under Unix/Linux.

Python MegaWidgets

Pmw is just pure Python code, and so doesn't need to be installed as such, just copied into a place where Python can find it. A link to a download of a tarred and zipped archive is available on the Pmw webpage. You should be able to unpack the archive with the command:

tar -xvzf Pmw.1.2.tar.gz

This will give you a folder called Pmw To install Pmw, copy this folder into the site-packages folder of your python installation, as show below. This command applies to Python 2.3 as installed as a framework build - please change the version number (2.3) to whatever is suitable for your installation if it is different. If you've built your own python, just copy the Pmw folder into the lib/site-packages of your install tree.

cp -R Pmw /System/Library/Frameworks/Python.framework/Versions/2.3/lib/python2.3/site-packages

VTK

Most of the instructions for building under Unix/Linux should apply, but you will first need to ensure you have a compiler installed, which you can do by installing the MacOSX developer tools from the installation CD, which includes gcc.
