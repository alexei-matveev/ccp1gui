The CCP1 Graphical User Interface

All of the packages needed to get the CCP1GUI running are freely available on the web and many come pre-installed on most modern operating systems. A number of the packages are not "standard" and so are unlikely to be installed on your computer unless you put them there.

To get the CCP1GUI running on your system you can either download a file containing all of the additional packages required for your Operating System together with the CCP1GUI source code from our ftp site as described in the Easy Installation instructions, or you can download the latest CCP1GUI package from the CCP1GUI pages on sourceforge and install all of the additional packages yourself, as described in the sections below for Linux, Macintosh and Windows users.

Easy Installation
NB: Python, Tcl/Tk and Tkinter (the interface between Python and Tk/Tcl) should come pre-installed on most modern operating systems and the rest of these instructions assume that they are. If they are not, please read the sections below for Linux, Macintosh or Windows for instructions on how to get hold of them ( they all are freely available ).

To get the archive file containing the CCP1GUI and all of the packages it depends on, go the to ccp1gui ftp site. You should be able to do this with your browser, if not you can use a command-line ftp client. To do this open a terminal on Linux / Unix / MacOSX or fire up the Windows command-prompt and type:

ftp ftp.dl.ac.uk

When promted for a password enter anonymous and just hit enter when asked for a password. Then cd to the ccp1gui directory.

The packages in the ccp1gui directory are organised according to Operating System (OS), OS version and processor-type ( if applicable ). The individual packages are named according to their directory, together with the name ccp1gui_bundle and the suffix .tar.gz for Unix or .zip for windows. For example, if you are running Redhat Linux 9.0 on an i586 ( i.e. pentium-type ) processor, you would download the file:

/qcg/ccp1gui/redhat/9.0/i586/redhat-9.0-i586-ccp1gui_bundle.tar.gz

If you are running Windows XP ( which will be running on a Pentium processor ), the file would be:

/qcg/ccp1gui/windows/xp/windows-xp-ccp1gui_bundle.zip

Once you have downloaded the file, you will need to upack it. To do this on unix systems, use the tar and gzip utilties ( see the Linux Help section for instructions on how to do this ) or use the Winzip utility on Windows.

The file will unpack to give a folder called ccp1gui_bundle, which has the following structure:


                  |--data/
ccp1gui_bundle/---|--ccp1gui.sh ( or ccp1gui.bat )
                  |--Numeric/
		  |--Numeric.pth
		  |--Scientific/
		  |--Pmw/
		  |--vtk/
		  |
		  |           |-- basis/
		  |           |-- ccp1gui.sh
		  |--ccp1gui/-|-- chempy/
		              |-- CVS/
			      |-- doc/------|--html/
			      |-- examples/
			      |-- generic/
			      |-- idle/
			      |-- interfaces/
			      |-- jobmanager/
			      |-- LICENCE
			      |-- objects
			      |-- scripts
			      |-- viewer

The script ccp1gui.sh on Unix/Linux/MacOSX systems and ccp1gui.bat on Windows systems in the ccp1gui_bundle directory should be used to start the CCP1GUI.

NB: Mac OSX users will need to run the script setpaths.sh before trying to run the CCP1GUI, as otherwise the paths to the VTK libraries are likely to be wrong.

On Unix and Linux systems, make sure the script is executable with the command:

chmod +x ccp1gui.sh

and then run it with the command:

./ccp1gui.sh

On Windows, just double-click on the ccp1gui.bat file.

Upgrading

If you would like to upgrade your version of the CCP1GUI ( if you hear of any new bug fixes or additional features you would like to take advantage of ), you will need to download a new ccp1gui directory to replace your current version.

The latest version of this directory is available for download from sourceforge or as the file ccp1gui-latest.tar.gz on our ftp site.

Remove ( or rename ) the old ccp1gui directory, download the new file into your ccp1gui_bundle directory and unnpack it so that the new directory ccp1gui replaces the old one. You can then run the ccp1gui.sh or ccp1gui.bat script as before.

Installing the packages individually

Linux

Before following the instructions below, you should query the package management for your Linux distribution to see if these packages are already available to you ( as it is likely to make life a lot easier for you ). The additional packages you will need to ensure are on your system are:

  • Tcl ( Tool Command Language )
  • Tk ( Tk Toolkit )
  • Tkinter ( aka python-tk )
  • Numeric Python ( aka python-numeric or numpy )
  • Scientific Python ( aka python-scientific )
  • Python Megawidgets ( aka Pmw )

The majority of these should be available under yast with Suse, apt-get with Debian, yum under Redhat or emerge under Gentoo. If they are not ( as is likely to be the case for Pmw at least ) read the instructions below on how to install them.

Even if the packages aren't available through your package manger, then you may be able to find rpm distributions of the relevant packages ( if you are running Redhat, Suse or their ilk ). A good place to start looking for rpm packages is the rpmfind site.

If can't find all of the packages mentioned above then you will need to install them manually as described below.

NB: The following are only general instructions that were correct at the time of writing. Software being what it is however, they are likely to become out of date fairly quickly. In all cases however, there should be up-to-date instructions on how to install the packages on their websites or included as README files in any downloads. The notes here should only be taken as a guide and you are strongly advised to read the documentation supplied with the packages to determine the best way to install them.

Python

There are are a range of different python distributions in use: python 1.5, 2.2, 2.3 and 2.4. The CCP1GUI should work quite happily with most of these.

If for some unaccountable reason Python isn't installed on your system, it is available as source from: www.python.org/download/

The CCP1GUI also relies on the additional Python packages: Numeric Python, Scientific Python and Python Mega Widgets (PMW).

Numeric Python

Numeric python is available from: http://sourceforge.net/projects/numpy

On this page, follow the link to numpy in the Latest Files Releases section and download a Numeric-<version_number>.tar.gz file to a folder on your computer.

To install the package, you will need to be in the directory you have just unpacked and have root priviledges. Now enter:

python setup.py install

This runs the setup.py python script that copies the relevant files into the path of the python version you are using to invoke the script.

Scientific Python

Scientific python is available from: http://starship.python.net/~hinsen/ScientificPython.

Both rpm and .tar.gz versions are available.To install the tar.gz version, unzip and untar the archive and then cd into the directory that is created and enter:

python setup.py install
Python Megawidgets (Pmw)

The final step to get the python side of things working is to install the Python Mega Widgets package.

This can be downloaded from: http://pmw.sourceforge.net/ as a gzipped tar archive.

Detailed installation instructions are available at: http://pmw.sourceforge.net/doc/starting.

To install the package unzip and untar the package. This will create a directory containing all of the PMW files and this then needs to be moved into a place where python will find it. Assuming you are running python 2.n ( where n is one of 1, 2, 3 or 4 depending on which version of python you are using ) a good place to put it is:

/usr/lib/python2.n/site-packages/

You can move the directory into this path by becoming root and entering:

mv <PMW_directory> /usr/lib/python2.2/site-packages/<PMW_directory>
Tkinter

Your Python distribution should come with a valid version of Tkinter (this may also be called python-tk). This is the python module that acts as the interface to the Tk toolkit (see below). If you mix and match your versions of Tcl/Tk and VTK that you may encouter problems with the version of Tkinter you are using. If this is the case, an error will be displayed indicating that a particular version of Tkinter is required when you try and run the CCP1GUI for the first time. If this is the case, you can search for an appropriate rpm package at: www.rpmfind.net/linux/RPM/. If you are unsure how to query and install rpm packages, please read the Linux Help section.

Tcl/Tk

Tcl (Tool Command Language) is an open source scripting language that has a simple and programmable syntax and can be either used as a standalone application or embedded in application programs. Tk is a graphical user interface toolkit used to create GUIs.

The latest version of Tcl/Tk is available as source code or with links to the pre-compiled binaries at:http://dev.scriptics.com/software/tcltk

As ever, rpm distributions can be found at: www.rpmfind.net/linux/RPM/

VTK

If you are fortunate then you may find that a vtk package already exists for your distribution. This is certainly the case for Debian, and if are running Redhat 9.0, rpm distributions for are available from:www.creatis.insa-lyon.fr/vtk/

If you are running Redhat 7.1, then vtk rpms for this distribution are available from:

www.fysik.dtu.dk/CAMPOS/Download/CAMPOS_access.html

The packages you will need to download from this site are:

  • vtk-<release_and_cvs_number>.i386.rpm
  • vtk-python-<release_and_cvs_number>.i386.rpm

If the above fails, then there are instructions on building VTK here, however, before you read these instructions, you will need to ensure that you have the developmental pacakges for Tk, Tcl and Python installed. The rpm versions of these packges are called:

  • python-devel-<version_number>
  • tcl-devel-<version_number>
  • tk-devel-<version_number>

where <version_number> is a version number compatible with whatever version of python, tk and tcl you are running.

Windows

The following steps describe two ways to get the CCP1GUI up and running on a Windows XP system. Both methods rely on the now outdated Python2.1 distribution, as an executable installer for VTK isn't available yet for the newer versions of Python. The first section of this documentation therefore explains how to install Python2.1 and PythonWin on a Windows XP System. It is possible to get the CCP1GUI to work with later versions of Python and VTK, although the installation procedure is a little more involved.

If you don't already have it, you will need to download Winzip. Although windows has its own 'zip' utility it cannot deal with .tar.gz archives. An evaluation version of Winzip can be downloaded from www.winzip.com.

Python2.1

First, download the Python-2.1.3.exe installer from: www.python.org. The uninstaller for this version of Python has a bug in it, so it's advisable to also download the UNWISE.exe file.

To install the program, just double-click on the executable icon. You will asked where to install Python and should accept the default option of: C:\Python21. It's also advisable to select the make backups option in case anything goes wrong. Once the installer has finished, over-write the UNWISE.exe file in the C:\Python21 directory with the one you downloaded.

PythonWin

The next stage is to download and install PythonWin for Python 2.1. The win32all-151.exe executable is available from http://starship.python.net/crew/mhammond/win32/Downloads.html. Just double-click the executable to install it.

The Visualisation Toolkit (VTK)

This package provide most of the graphical functionality of the CCP1GUI. A windows executable that is compatible with Python 2.1 is available from the VTK website.

Go to the Wintel Pre-Compiled Binaries (Windows 9x/NT) section and download the file vtk42-LatestRelease.exe.

To install VTK, just run the vtk42-LatestRelease.exe executable, and it will install VTK in C:\Program Files\vtk42.

Display problems with Windows NT

We have found that on our Windows NT systems it has been necessary to change the number of colours (choose Start -> Settings -> control panels -> Display) i.e. 1677216) - with the default the atom picking didn't work properly.

Macintosh OSX

Unfortunately, the version of Python that ships with OSX (Darwin at any rate) does not include Tk/Tcl or Tkinter by default (why?!?), so you will need to install these packages first.

Tcl/Tk Aqua

Tcl/Tk for MacOSX can be downloaded from Apple - just follow the instructions for installing them.

Tkinter ( Numeric Python and IDLE )

The next step is to install the Mac Python add-ons, which include a Package Manager tool that you can use to install Tkinter ( as well as Numeric Python and IDLE ). You can get the Package Manger from MacPython website. Once you have installed this, it will create a folder call MacPython-version ( where version is the version of Python you are using - e.g. 2.3 ) in your Applications folder. Use the Finder to navigate to this folder and then run PackageManager program.

NB: If this crashes on opening with a message about being unable to open a ".plist" file, then the Package Manager is looking at the wrong online database to get it's list of available packages. To find out about current databases, visit the packman webpage and get the url for a suitable database. Then in PackageManager, select File -> Open Url... and paste in the url.

In the package manager then select the following packages:

  • Numeric-version-binary
  • _tkinter-version-binary
  • IDLE-version-binary

( where version will be a differnent number depending on the version ) and then click the install button at the bottom of the page to install the packages.

If the above commands work for you, you should now be in a position to download one of the pre-packaged distributions or Mac OSX as described in the Easy Installation Instructions above. Alternatively, you can install the remaining dependencies as described below:

Scientific Python and Pmw

The instructions for installing Scientific Python and Pmw under Unix/Linux are also applicable under Mac OSX, so please read the information in those sections.

VTK

If you have installed Tk/Tcl and Tkinter as described above, then follow the instructions below to build vtk.

Building VTK from source

The Visualization ToolKit (VTK) is an open source, freely available software system for 3D computer graphics, image processing, and visualization that can be used or creating graphical representations of molecules.

The following instructions are for building VTK from source. To do this you will need a C/C++-compiler. On Linux systems gcc will be there by default, on Macs, you can install fink and use this to install gcc. On Windows, your best bet is probably to install cygwin and get gcc that way.

With a compiler in place you will need to install the relevant development packages on your system ( development packages contain the header files for C/C++ and other information needed when you compile a package ). Assume you already have Tk/Tcl, Python and Tkinter installed as mentioned in the sections above, then you will just need to install the development versions of these packges.

The next stage is to install cmake, the cross-platform make utility. You can get this from the CMake website.

Next, download the lastest VTK release from http://public.kitware.com/VTK/get-software.php. The file will extract to a directory called VTK. cd into this directory and enter:

cmake -i

This runs CMake in interactive mode, where it will ask you a number of different questions. The first question you will be asked is if you would like to see the advanced options - if you do you probably don't need to be reading this. Most of the remaining options shouldn't need to be changed and those that you may want to should be self-explanatory. Options that you must select (i.e. set to ON), however are the following:

  • BUILD_SHARED_LIBS
  • VTK_USE_HYBRID
  • VTK_WRAP_PYTHON
  • VTK_WRAP_TCL

When CMake has finished, just type:

make

in the VTK directory to build the VTK binaries, and then:

make install

To install vtk onto your system.
