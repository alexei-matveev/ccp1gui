The CCP1 Graphical User Interface

The Graphical User Interface (GUI) for CCP1 is built using a number of different technologies. The 'interface', including things like the menus and buttons, are implemented using the Tcl/Tk and PMW toolkits. The drawing of molecules on the screen is carried out by the Visualisation Toolkit (VTK) using the OpenGl graphics standard. The 'glue' that links all of these technologies together to present a single coherent interface is the Python programming language.

Python, Tcl/Tk, VTK and OpenGL are available for almost every flavour of Linux, Macintosh or Windows, so CCP1 can be made to run on any of these systems. The following sections describe the steps you will need to take to install the packages needed to get the CCP1GUI running on your system.

Linux

There are two ways to get the CCP1GUI up and running on your system. The first is the easiest and least-involved, although it will require that you have a reasonably fast internet connection as the file to be downloaded is about 20MB in size. The second is more involved and should only be followed if you know what software you have installed on your system and are happy installing new packages.

Easy Installation

Most modern Linux distributions ship with Python, Tk/Tcl and Tkinter (aka python-tk) already installed. If you are missing any of these packages, the Python, Tcl and Tkinter sections below will guide you through their installation and the Linux Help section, provides general help on the ins and outs of installing software under Linux.

With Python and Tcl installed, all you need to do is download the ccp1gui-<version>.tar.gz file suitable for your Linux distribution. The files are available via ftp from ftp://ftp.dl.ac.uk/qcg/ccp1gui - just follow the link to your distribution. Currently we have packaged distributions for Redhat 9.0 and Suse 9.0.

Place the file in a directory somewhere on your hard drive and unpackage it with the command:

tar -xvzf ccp1gui-<version>.tar.gz

where <version> is a number indicating the release of the CCP1GUI with rh9 appended to it for Redhat9.0 and suse9 for Suse 9.0.

The file will un-tar into a directory called ccp1gui. The directory has the following structure:


           |--data/
ccp1gui/---|--ccp1gui.bash
           |--ccp1gui.bat
           |--VTK/
           |--Numeric/
           |--Numeric.pth
           |--Scientific/
           |--Pmw
           |
           |              |-- basis/
           |              |-- ccp1gui/
           |--python_dev/-|-- ccp1gui.bat
                          |-- ccp1gui.sh
                          |-- chempy
                          |-- examples/
                          |-- graph/
                          |-- jobmanager/
                          |-- pymolgraph/
                          |-- README
                          |-- tests
                          |-- tkmolview
                          |-- doc/------|--html/
To run the CCP1GUI, just cd into this directory and run the ccp1gui.bash script with the command:

./ccp1gui.bash

Upgrading

If you would like to upgrade your version of the CCP1GUI, if you hear of any new bug fixes or additional features, you will need to download a new python_dev directory to replace your current version.

The latest version of this directory is always available on our ftp site. Download the file ccp1gui-<latest_release>.tar.gz into your ccp1gui directory, and unpack it with the command:

tar-xvzf ccp1gui-<latest_release>.tar.gz

where <latest_release> is the release version you have downloaded. This will unpack to create a new python_dev directory. You should then be able to run the CCP1GUI using the ccp1gui.bash script in the ccp1gui directory as before.

If you have any problems installing or running the CCP1GUI using this method, please let us know.

Full Installation

This section describes the seperate instalation steps for all of the individual pieces of software that are used by the CCP1GUI. If you are unfamiliar with the basics of how to install software distributed as rpm files or zipped and compressed archives, there are brief notes in the Linux Help section.

These are only general instructions that were correct at the time of writing. Software being what it is however, they are likely to become out of date fairly quickly. In all cases however, there should be up-to-date instructions on how to install the packages on their websites or included as README files in any downloads. The notes here should only be taken as a guide and you are strongly advised to read the documentation supplied with the packages to determine the best way to install them.

Python

There are currently three different python distributions in use: python 1.5, 2.2 and 2.3. Of these, python 2.2 is the most common.

If you have more than one python distribution installed (as may well be the case if you are running Redhat) you will need to ensure that you invoke the correct version whenever you install a python-related package. A particular version of python will install any new packages within its own paths which may not be accessible to other distributions.

To find out what executable you are calling when you type python, enter:

which python

This will show the path to the executable you are calling on when you type python. List the contents of that directory to see if you have more than one version of python there (e.g. /usr/bin/python and /usr/bin/python2). If you do, you will need to decide which version to use (python 2.2 is recommended) and stick with it - in the above case by typing python2 whenever you invoke python.

Python is available as source from:www.python.org/download/.

You can search for an appropriate rpm package at: www.rpmfind.net/linux/RPM/ .

The CCP1GUI also relies on the additional Python packages: Numeric Python, Scientific Python and Python Mega Widgets (PMW). These packages are not usually installed as rpms and are not part of the standard python distribution, so are unlikely to have been installed on your system.

Numeric Python

Numeric python is available from: http://sourceforge.net/projects/numpy.

On this page, follow the link to numpy in the Latest Files Releases section and download a Numeric-<version_number>.tar.gz file to a folder on your computer.

To install the package, you will need to be in the directory you have just unpacked and have root priviledges. Now enter:

python setup.py install

This runs the setup.py python script that copies the relevant files into the path of the python version you are using to invoke the script.

Scientific Python

Scientific python is available from: http://starship.python.net/~hinsen/ScientificPython.

Both rpm and .tar.gz versions are available.To install the tar.gz version, unzip and untar the archive and then cd into the directory that is created and enter:

python setup.py build

This will build the relevant files which will then need to be installed. Installing the files will probably need root permission, so su to root and then enter:

python setup.py install

Python Megawidgets (Pmw)

The final step to get the python side of things working is to install the Python Mega Widgets package.

This can be downloaded from: http://pmw.sourceforge.net/ as a gzipped tar archive.

Detailed installation instructions are available at: http://pmw.sourceforge.net/doc/starting.

To install the package unzip and untar the package. This will create a directory containing all of the PMW files and this then needs to be moved into a place where python will find it. Assuming you are running python 2.2 a good place to put it is:

/usr/lib/python2.2/site-packages/

You can move the directory into this path by becoming root and entering:

mv <PMW_directory> /usr/lib/python2.2/site-packages/<PMW_directory>

If you can't get root priviledges, or you don't want to move the directory, it is also possible put the path for the PMW directory in your PYTHONPATH variable. The PYTHONPATH variables tells python where to look for the files it needs. To append the Pmw directory to your existing PYTHONPATH enter:

export PYTHONPATH=$PYTHONPATH:<Pmw_directory>

Where <Pmw_directory> is the full path to the Pmw directory.

Tkinter

Your Python distribution should come with a valid version of Tkinter (this may also be called python-tk). This is the python module that acts as the interface to the Tk toolkit (see below). If you mix and match your versions of Tcl/Tk and VTK that you may encouter problems with the version of Tkinter you are using. If this is the case, an error will be displayed indicating that a particular version of Tkinter is required when you try and run the CCP1GUI for the first time. If this is the case, you can search for an appropriate rpm package at: www.rpmfind.net/linux/RPM/. If you are unsure how to query and install rpm packages, please read the Linux Help section.

Tcl/Tk

Tcl (Tool Command Language) is an open source scripting language that has a simple and programmable syntax and can be either used as a standalone application or embedded in application programs. Tk is a graphical user interface toolkit used to create GUIs.

The latest version of Tcl/Tk is available as source code or with links to the pre-compiled binaries at:http://dev.scriptics.com/software/tcltk

As ever, rpm distributions can be found at: www.rpmfind.net/linux/RPM/

VTK

The Visualization ToolKit (VTK) is an open source, freely available software system for 3D computer graphics, image processing, and visualization that can be used or creating graphical representations of molecules.

VTK can be installed from source, or there are rpm distributions available for Redhat9.0 and Redhat7.3 (see below).

If you decide to install VTK from source, you will firstly need to download the CMake cross-platform make utility.

You will also need to ensure that you have all of the developmental software for Tk, Tcl and Python installed. The rpm versions of these packges are called:

  • python-devel-<version_number>
  • tcl-devel-<version_number>
  • tk-devel-<version_number>

where <version_number> is a version number compatible with whatever version of python, tk and tcl you are running.

With these packages installed, download the lastest VTK release from http://public.kitware.com/VTK/get-software.php. The file will extract to a directory called VTK. cd into this directory and enter:

cmake -i

This runs CMake in interactive mode, where it will ask you a number of different questions. The first question you will be asked is if you would like to see the advanced options - if you do you probably don't need to be reading this. Most of the remaining options shouldn't need to be changed and those that you may want to should be self-explanatory. Options that you must select (i.e. set to ON), however are the following:

  • BUILD_SHARED_LIBS
  • VTK_USE_HYBRID
  • VTK_WRAP_PYTHON
  • VTK_WRAP_TCL

When CMake has finished, just type:

make

in the VTK directory to build the VTK binaries, and then:

make install

if you wish to install the files.

VTK rpm's

If all of the above sounds a little involved and you are running Redhat 9.0, more manageable rpm distributions for are available from:www.creatis.insa-lyon.fr/vtk/

If you are running Redhat 7.1, then vtk rpms for this distribution areavailable from:

www.fysik.dtu.dk/CAMPOS/Download/CAMPOS_access.html

The packages you will need to download from this site are:

  • vtk-<release_and_cvs_number>.i386.rpm
  • vtk-python-<release_and_cvs_number>.i386.rpm

Getting the CCP1GUI to work

Once you have prepared your system by installing the above packages, you are ready to download and run the CCP1GUI. Download the latest version of the CCP1GUI from our ftp site at ftp://ftp.dl.ac.uk/qcg/ccp1gui/ccp1gui-<latest_release>.tar.gz and extract the archive to a directory called python_dev on your hard drive. Before you run the CCP1GUI you will need to add the path to the python_dev directory to your PYTHONPATH variable. To do this enter:

export PYTHONPATH=$PYTHONPATH:<directory_prefix>/python_dev

where directory_prefix is the full path to the python_dev directory. To get the CCP1GUI to run, you need to run python on the vtkgraph.py file in the python_dev/tkmolview directory. To do this enter:

python <directory_prefix>/python_dev/tkmolview/vtkgraph.py

NB. This assumes that the python runs the correct version of python as was mentioned in the python section.

Windows

The following steps describe two ways to get the CCP1GUI up and running on a Windows XP system. Both methods rely on the now outdated Python2.1 distribution, as an executable installer for VTK isn't available yet for the newer versions of Python. The first section of this documentation therefore explains how to install Python2.1 and PythonWin on a Windows XP System. It is possible to get the CCP1GUI to work with later versions of Python and VTK, although the installation procedure is a little more involved.

If you don't already have it, you will need to download Winzip. Although windows has its own 'zip' utility it cannot deal with .tar.gz archives. An evaluation version of Winzip can be downloaded from www.winzip.com.

Python2.1

First, download the Python-2.1.3.exe installer from: www.python.org. The uninstaller for this version of Python has a bug in it, so it's advisable to also download the UNWISE.exe file.

To install the program, just double-click on the executable icon. You will asked where to install Python and should accept the default option of: C:\Python21. It's also advisable to select the make backups option in case anything goes wrong. Once the installer has finished, over-write the UNWISE.exe file in the C:\Python21 directory with the one you downloaded.

PythonWin

The next stage is to download and install PythonWin for Python 2.1. The win32all-151.exe executable is available from http://starship.python.net/crew/mhammond/win32/Downloads.html. Just double-click the executable to install it.

Easy Installation of the CCP1GUI

There is a version of the GUI available for FTP download, that packages all of the files required to run the CCP1GUI into a single directory. The directory structure is the same as that for the Linux distribution shown above. The only additional requirements for this version are Python 2.1 and PythonWin.

Once you have these installed, download the file ccp1gui-<version>_win.tar.gz file from our ftp site and uncompress the package using WinZip. The package will decompress to give a directory called ccpigui. To run the CCP1GUI, open the ccp1gui directory and double-click on the ccp1gui.bat batch file. If it doesn't work

If for some reason the GUI doesn't run when you click on the batch file, the most likely reason is that it cannot find all of the files it needs to run. The following section describes how the batch file sets up the 'environment' that the CCP1GUI runs in, so that you can see if this is correct for your system and change it if it isn't.

The batch file is displayed below (the purple line numbers are just there to aid explanation and don't appear in the file proper):

1.     %0\

2.     cd %0\.

3.     set ROOTDIR =%CD%

4.     set PATH=%PATH%; %ROOTDIR%\vtk42\bin

5.     set PYTHONPATH=%PYTHONPATH%; %ROOTDIR%\python_dev

6.     set PYTHONPATH=%PYTHONPATH%; %ROOTDIR%\vtk42\Wrapping\Python

7.     set PYTHONPATH=%PYTHONPATH%; %ROOTDIR%\vtk42\bin

8.     set PYTHONPATH=%PYTHONPATH%; %ROOTDIR%\Numeric

9.     set PYTHONPATH=%PYTHONPATH%; %ROOTDIR%

10.   c:\python21\python.exe " %ROOTDIR%\python_dev\tkmolview\vtkgraph.py" %1

In a batch file, the argument %0\ refers to the batch file itself, so the first two lines ensure that the CCP1GUI is run from the main installation directory. Line 3 then sets the variable ROOTDIR to signify this directory (this is done so that all other directory paths are relative to this directory, so you can install the CCP1GUI whereever you wish).

Line 4 appends the directory containing the VTK library files to the PATH environment variable . If you have installed your VTK library files elseswhere, you will need to change this.

Lines 5 to 9 build up the PYTHONPATH environment variable, which tells python where to find the files it needs. If you have installed VTK or Numeric or any of the other modules somewhere else you will need to change the lines accordingly.

Line 10 runs the CCP1GUI using Python version 2.1, which it expects to find installed in the folder C:\Python21 If you have installed Python2.1 elsewhere, then change this line.

Alternatively, to run the CCP1GUI using Pythonwin, open Pythonwin (Start -> Programs -> Python2.1 -> Pythonwin) and go to File -> Run.... Click on the Browse button and select the vtkgraph in the python_dev/tkmolview directory and click Open and then click on Ok. For this to work, you will have to have set your PYTHONPATH variable accordingly.

Display problems with Windows NT

We have found that on our Windows NT systems it has been necessary to change the number of colours (choose Start -> Settings -> control panels -> Display) i.e. 1677216) - with the default the atom picking didn't work properly.

Setting the PYTHONPATH environment variable.

To set the PYTHONPATH variable go to Control Panel -> System-> Advanced and click the 'Environment Variables' button. In the 'System Variables' panel that appears, click New and enter PYTHONPATH in the 'Variable Name' field. In the 'Variable Value' field enter the string corresponding to the path of the directories you wish to include in your PYTHONPATH. For the CCP1GUI this should include the following paths:

  • <python_prefix>\python_dev
  • <vtk_prefix>\vtk42\Wrapping\Python
  • <vtk_prefix>\vtk42\bin

where <python_prefix> is the path to your python_dev directory, and <vtk_prefix>, the path to wherever you have installed vtk.

If your operating system is installed on your C: drive and you have put all the programs in their default locations, your PYTHONPATH will be the following:

c:\python_dev;c:\Program Files\vtk42\Wrapping\Python;c:\Program Files\vtk42\bin
