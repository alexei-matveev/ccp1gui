Installation of the packages for the CCP1GUI under Unix/Linux

SuSE

Under OpenSuse, the following rpm files will need to installed:

  • python-tk
  • vtk-python

In addition, to access the full functionality of the CCP1GUI you should also install:

  • python-numpy
  • python-scipy

Ubuntu

Under Ubuntu, the following packages need to installed:

  • python-tk
  • python-vtk
  • python-pmw
  • python-numeric
  • python-numeric-ext

These packages can be installed using apt-get or synaptic if the Universe repositories have been enabled. For advice on how to do this see:

http://help.ubuntu.com/community/Repositories/Ubuntu

Manual installation of additional Packages

You should only follow these instructions if you cannot get the packages via the package management system of your operating system.

Numerical Python

The Scientific Python website has a download link for the recommended version of Numerical Python, together with instructions on how to install it.

Scientific Python

The Scientific Python website has a download link and installation instructions for Scientific Python.

Pmw

Pmw is just pure Python code, and so doesn't need to be installed as such, just copied into a place where Python can find it. A link to a download of a tarred and zipped archive is available on the Pmw webpage. You should be able to unpack the archive with the command:

gunzip < Pmw.1.2.tar.gz | tar xvf -

This will give you a folder called Pmw To install Pmw, copy this folder into the site-packages folder of your python installation, as show below. This command applies to Python 2.3 as installed in /usr/lib. Please change the version number (2.3) and prefix to whatever is suitable for your installation if it is different.

cp -R Pmw /usr/lib/python2.3/site-packages/

NB:If you aren't sure where you python site-packages is located, you can find out by executing the following sequence of commands:

python

This should start the python interpreter on your system and give you a python prompt (something like >>> should appear). If that does, type:

import sys

This imports the system module that we can then query to find the python path with the command:

print sys.path

This will print out the path that Python searches for Modules. Look for the one that has site-packages in it.

VTK

The Visualization ToolKit (VTK) is an open source, freely available software system for 3D computer graphics, image processing, and visualization that can be used for creating graphical representations of molecules.

The following instructions are for building VTK from source.

With a compiler in place you will need to install the relevant development packages on your system ( development packages contain the header files for C/C++ and other information needed when you compile a package ). Assuming you already have Tk/Tcl, Python and Tkinter installed as mentioned in the sections above, then you will just need to install the development versions of these packges.

The next stage is to install cmake, the cross-platform make utility. You can get this from the CMake website.

Next, download the lastest VTK release from http://public.kitware.com/VTK/get-software.php. The file will extract to a directory called VTK. cd into this directory and enter:

ccmake

This runs CMake with a curses interface, where it will ask you a number of different questions. The first question you will be asked is if you would like to see the advanced options - if you do you probably don't need to be reading this. Most of the remaining options shouldn't need to be changed and those that you may want to should be self-explanatory. Options that you must select (i.e. set to ON), however are the following:

  • BUILD_SHARED_LIBS
  • VTK_USE_HYBRID
  • VTK_WRAP_PYTHON
  • VTK_WRAP_TCL

When CMake has finished, just type:

make

in the VTK directory to build the VTK binaries, and then:

make install

to install vtk onto your system.
