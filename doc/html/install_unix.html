Installation of the packages for the CCP1GUI under Unix/Linux

Base Packages

The majority of these should be available under the package management system under Linux systems (e.g. yast with Suse, apt-get with Debian or Ubuntu, yum under Redhat or emerge under Gentoo) or on the installation cd's for your distribution. Even if they are not, then you may be able to find rpm distributions of the relevant packages ( if you are running Redhat, Suse or their ilk ).

If you are unfamiliar with Unix/Linux command-line tools and how to determine which rpms are installed / how to add new packages, there are some instructions on how to use these in the Linux Help section.

A good place to start looking for rpm packages is the rpmfind site.

Python

There are a range of different python distributions in use: Python 1.5, 2.2, 2.3 and 2.4. The CCP1GUI should work quite happily with most of these, although you are recommended to try and get the most up-to-date possible.

If Python isn't installed on your system, it is available from: www.python.org/download/

Tcl/Tk

A good place to look for Tcl/Tk packages is the Tcl Developer Xchange, or you can always get it as source from the Tcl/Tk pages on Sourceforge, by clicking on the File Distribution link.

Tkinter

Tkinter is really considered to be part of Python and so is usually included with most Python distributions (and comes if you download Python as source code). Unfortunately, many Linux distributions do not install Tkinter by default, even though it is provided, so you will need to look for the appropriate rpm on the installation cds or via the package manager. Additional information on Tkinter can be found at: http://wiki.python.org/moin/TkInter.

With the three above packages installed, you should be in a position to download a bundle for your system. If one is not available, then please read on for instructions on how to install the additional packages.

Additional Packages

Numerical Python

The Numerical Python website at sourceforge has links to tarred and gzipped archives that you can find by clicking on the Download Numerical Python link. Then select the numpy package to download and you will be presented with a page that offers rpm packages as well as source distributions.

If you are using the source distribution, you will need to unpack the distribution with the command:

gunzip < Numeric-24.2.tar.gz | tar xvf -

To install the package, you will need to cd into the Numeric-24.2 you have just unpacked and have root priviledges. Whilst sat in the directory Numeric-24.2 enter the command:

python setup.py install

This runs the setup.py Python script that copies the relevant files into the path of the Python version you are using to invoke the script. If you want to install the python somewhere else, the you can find more information on how to use setup.py scripts at the distutils page on the Python website.

Scientific Python

The Scientific Python website: http://dirac.cnrs-orleans.fr/plone/software/scientificpython/ has links to downloads for rpm files as well as source distributions.

If you are using the source distribution, you will need to unpack the distribution with the command:

gunzip < Scientific-2.4.9.tar.gz | tar xvf -

To install the package, you will need to cd into the Scientific-2.4.9 you have just unpacked and have root priviledges. Whilst sat in the directory Scientific-2.4.9 enter the command:

python setup.py install

This runs the setup.py Python script that copies the relevant files into the path of the Python version you are using to invoke the script. If you want to install the Python somewhere else, the you can find more information on how to use setup.py scripts at the distutils page on the Python website.

Pmw

Pmw is just pure Python code, and so doesn't need to be installed as such, just copied into a place where Python can find it. A link to a download of a tarred and zipped archive is available on the Pmw webpage. You should be able to unpack the archive with the command:

gunzip < Pmw.1.2.tar.gz | tar xvf -

This will give you a folder called Pmw To install Pmw, copy this folder into the site-packages folder of your python installation, as show below. This command applies to Python 2.3 as installed in /usr/lib. Please change the version number (2.3) and prefix to whatever is suitable for your installation if it is different.

cp -R Pmw /usr/lib/python2.3/site-packages/

NB:If you aren't sure where you python site-packages is located, you can find out by executing the following sequence of commands:

python

This should start the python interpreter on your system and give you a python prompt (something like >>> should appear). If that does, type:

import sys

This imports the system module that we can then query to find the python path with the command:

print sys.path

This will print out the path that Python searches for Modules. Look for the one that has site-packages in it.

VTK

If you are fortunate then you may find that a vtk package already exists for your distribution. This is certainly the case for Debian, and if are running Redhat 9.0, rpm distributions for are available from: www.creatis.insa-lyon.fr/vtk/

If the above fails, then the following instructions are for building VTK.

The Visualization ToolKit (VTK) is an open source, freely available software system for 3D computer graphics, image processing, and visualization that can be used for creating graphical representations of molecules.

The following instructions are for building VTK from source. To do this you will need a C/C++-compiler. On Linux systems gcc will be there by default, on Macs, you can user the developer tools or install fink and use this to install gcc. On Windows, your best bet is probably to install cygwin and get gcc that way.

With a compiler in place you will need to install the relevant development packages on your system ( development packages contain the header files for C/C++ and other information needed when you compile a package ). Assuming you already have Tk/Tcl, Python and Tkinter installed as mentioned in the sections above, then you will just need to install the development versions of these packges.

The next stage is to install cmake, the cross-platform make utility. You can get this from the CMake website.

Next, download the lastest VTK release from http://public.kitware.com/VTK/get-software.php. The file will extract to a directory called VTK. cd into this directory and enter:

ccmake

This runs CMake with a curses interface, where it will ask you a number of different questions. The first question you will be asked is if you would like to see the advanced options - if you do you probably don't need to be reading this. Most of the remaining options shouldn't need to be changed and those that you may want to should be self-explanatory. Options that you must select (i.e. set to ON), however are the following:

  • BUILD_SHARED_LIBS
  • VTK_USE_HYBRID
  • VTK_WRAP_PYTHON
  • VTK_WRAP_TCL

When CMake has finished, just type:

make

in the VTK directory to build the VTK binaries, and then:

make install

to install vtk onto your system.

The CCP1GUI under Debian/Ubuntu

Using apt-get

To install the CCP1GUI on a Debian or Ubuntu system, you will need to install the following packages:

  • python-numeric
  • python-scientific
  • python-pmw
  • python-vtk

You can either use the synaptic package manager GUI to do this in Ubuntu, (in which case all the packages are listed under the "Python Programming Language (Universe)" section, or use the command:

sudo apt-get install python-numeric python-scientific python-pmw python-vtk

from the command line in both Debian and Ubuntu.
